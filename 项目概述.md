**“Time Capsule: 永恒交响（Eternal Symphony）”**

------

## 🚀 项目名称：Time Capsule - 永恒交响

**定位：** 基于 Next.js 15 + React 19 的交互式数字化回忆博物馆。

### 1. 核心创新点 (The "Wow" Factors)

- **多维时空轴（Multi-Dimensional Timeline）：** 打破传统的纵向瀑布流，采用 **“星云式”或“螺旋式”** 的 3D 时间轴。用户可以像在星空中航行一样，滑动滚轮进入不同的年份“星团”，每个星团里悬浮着日记和照片。

- **情感气泡（Emotion Bubbles）：**

  利用 React 19 的粒子动画，日记不再是枯燥的文字块，而是根据内容情感（如欢快、怀念）呈现不同颜色和跳动频率的“气泡”，点击后气泡破裂，展示内容。

- **AI 叙事辅助：**

  接入轻量化模型，当你上传一组照片时，AI 会自动提取照片中的颜色和元素，生成一段极具诗意的“引言”或背景音乐推荐。

- **数字遗产锁（Legacy Lock）：**

  不仅有普通登录，还可以设置“未来信件”。你可以写下一段话，设定在未来的某个特定日期（如 10 年后）才允许开启。

------

### 2. 核心功能模块

| **模块**                       | **详细描述**                                                 |
| ------------------------------ | ------------------------------------------------------------ |
| **时空影廊 (Media Gallery)**   | 支持瀑布流与 **Bento Grid（便签盒）** 布局，照片可挂载语音旁白。 |
| **心语瓶 (Digital Journal)**   | 模仿纸质手账的质感，支持 Markdown 排版，可添加实时天气和地理位置印章。 |
| **密闭阁楼 (Privacy Vault)**   | **双重加密系统**：除了账户登录，核心敏感分类（如私密日记）需输入二次独立密码或手势密码。 |
| **访客留言墙 (Memory Bridge)** | 如果网页设为半公开，亲友可以留言并“投递”一张老照片，需由管理员审核通过后才会出现在回忆轴上。 |

------

### 3. 技术实现方案 (基于你的技术栈)

- **表现层：** 使用 **Tailwind CSS + Framer Motion** 实现极其丝滑的转场动画（如页面像翻书一样翻开）。
- **交互逻辑：** 利用 **React 19 的 Server Components** 加载海量图片资源，确保首屏秒开；使用 **Suspense** 实现数据流式加载。
- **数据安全：** * **后端：** Next.js API Routes (Server Actions) 处理业务逻辑。
  - **加密：** 使用 `bcrypt` 处理账户密码，敏感内容可用 `AES-256` 在存入数据库前进行二次加密。
- **部署：** 部署在 Vercel，图片资源托管在 Supabase 或阿里云 OSS，保证访问速度。

------

### 4. 界面美学设计 (Visual Identity)

- **风格：** **新古典主义 (Neo-Brutalism) 与 极简主义 (Minimalism)** 的结合。
- **色彩：** 采用**低饱和度**的莫兰迪色系（如：灰豆绿、奶油白、冷调灰），在暗色模式下转为深邃的“午夜蓝”。
- **动效：** 所有的按钮交互都有微弱的振动反馈和毛玻璃（Backdrop Blur）效果，营造出一种“在梦境中漫步”的质感。

### 5. 具体的项目描述文本（可直接用于申报书）

> **项目名称：** “永恒交响”——多维交互式个人回忆馆

> **描述：** 本项目是一款基于 Next.js 15 与 React 19 的全栈式数字化回忆管理平台。不同于传统的社交软件，本项目专注于“私密性”与“纪念性”，通过创新的“星云式 3D 时间轴”与“情感气泡”交互技术，将碎片化的照片与日记重构成一段连续的生命乐章。系统通过 AES-256 二次加密技术保障用户数据的绝对私密，并引入“未来信件”功能，实现跨越时空的情感寄托。旨在为用户打造一个既能安放过去，又能对话未来的数字化沉浸式纪念空间。

### 6. 本地开发与数据库（解决「服务端数据库不可用」）

**→ 想快速在本地登录调试、不想每次推 GitHub：直接看根目录的 [本地调试.md](./本地调试.md)，按三步做一次即可。**

公网部署在 Vercel 时，数据库连接写在 Vercel 的环境变量里，所以线上注册/登录正常；**本地没有配置数据库时**会报「服务端数据库不可用：请检查 DATABASE_URL / 迁移 / 数据库服务」。

**处理步骤：**

1. **拿到和公网相同的数据库连接串**  
   在 Vercel 项目 → **Settings** → **Environment Variables** 里，复制 **DATABASE_URL**（若使用 Vercel Postgres / Neon 等，通常还有 **DIRECT_DATABASE_URL**，也一并复制）。

2. **在本地建 `.env.local`**  
   在项目根目录（与 `package.json` 同级）新建 `.env.local`，内容示例：
   ```env
   DATABASE_URL="postgresql://用户:密码@主机:5432/数据库名?schema=public"
   # 若迁移报错需要直连，再填下面这行
   DIRECT_DATABASE_URL="postgresql://..."
   ```
   可直接参考根目录下的 `.env.example`。

3. **安装依赖并生成 Prisma 客户端**  
   ```bash
   npm install
   npx prisma generate
   ```

4. **（若库是新建或未跑过迁移）执行迁移**  
   ```bash
   npx prisma migrate deploy
   ```
   若使用连接池且未配置直连，可先只设 `DIRECT_DATABASE_URL` 再执行上述命令。

5. **本地启动**  
   ```bash
   npm run dev
   ```
   之后在本地打开注册/登录页即可正常使用。
