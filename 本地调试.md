# 本地登录调试（一次配置，之后不用再推 GitHub）

页面上出现 **「服务端数据库不可用」** 是因为本地没有连上数据库。按下面做**一次**，以后在本地直接 `npm run dev` 就能登录调试。

---

## 第一步：从 Vercel 复制环境变量（只做一次）

1. 打开 [Vercel Dashboard](https://vercel.com) → 你的项目 → **Settings** → **Environment Variables**。
2. 找到 **DATABASE_URL**，点开复制**整段**连接串。
3. 若还有 **DIRECT_DATABASE_URL** 或 **POSTGRES_URL_NON_POOLING**，也复制一份。

---

## 第二步：在项目根目录建 `.env.local`（只做一次）

在项目根目录（和 `package.json` 同一层）新建文件 **`.env.local`**，内容：

```env
DATABASE_URL="这里粘贴刚才复制的 DATABASE_URL"
DIRECT_DATABASE_URL="这里粘贴直连地址（若没有，就再贴一遍 DATABASE_URL）"
```

保存后，**不要**把 `.env.local` 提交到 Git（已在 `.gitignore` 里）。

---

## 第三步：让本地数据库就绪（只做一次）

在项目根目录打开终端，执行：

```bash
npm run local:db
```

会执行 `prisma generate` 和 `prisma migrate deploy`。若报错，多半是连接串不对或没填 `DIRECT_DATABASE_URL`，把 Vercel 里两个都复制过来再试。

---

## 之后每次本地调试

在项目根目录执行：

```bash
npm run dev
```

浏览器打开 `http://localhost:3000`，进登录/注册页即可，**无需再推 GitHub**。本地和线上用的是同一套 Vercel Postgres 数据。

---

## 不想配数据库时：用本地虚拟登录

在本地运行 `npm run dev` 时，可以用**虚拟账号**直接登录，不连数据库：

- **邮箱：** `demo@local`
- **密码：** `demo123`

仅开发环境有效，公网部署不会接受该账号，不影响线上安全。

**说明：** 演示账号只能浏览页面，**不能创建/编辑年份星团等数据**（会提示“本地演示账号仅可浏览…”）。要体验创建功能请在公网注册并登录。

---

## 若仍提示「服务不可用 / 数据库不可用」

1. **改过 `.env.local` 后一定要重启**  
   在运行 `npm run dev` 的终端里按 `Ctrl+C` 停掉，再重新执行 `npm run dev`。Next.js 只在启动时读取环境变量，不重启不会生效。

2. **确认 `.env.local` 内容**  
   两行都要填，且不能有多余空格或断行：
   ```env
   DATABASE_URL="postgresql://..."
   DIRECT_DATABASE_URL="postgresql://..."
   ```
   若只有一个连接串，两行都填同一个即可。

3. **确认在项目根目录启动**  
   终端当前目录应是 `D:\Desktop\网页开发`（和 `package.json`、`.env.local` 同一层），否则可能读不到 `.env.local`。
